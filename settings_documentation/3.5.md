# Промпт для пункта: 3.5

## Заголовок

**3.5 Рекомендации по использованию**

## Промпт для нейросети

Напиши раздел отчета с заголовком **«3.5 Рекомендации по использованию»**.

Полностью адаптируй содержание под тему **«Разработка информационной системы электронного расписания для учебного заведения»**.  
Не используй формулировки про интернет-магазин, онлайн-курсы, корзину, оплату и контроль успеваемости. Не упоминай мессенджер Telegram; вместо него используй слово «мессенджер».

### Содержание раздела

**Вводная часть.** Укажи, что рекомендации предназначены для корректной эксплуатации информационной системы электронного расписания и описывают рациональный порядок работы для разных категорий пользователей. Подчеркни, что это не полное руководство пользователя, а практические рекомендации.

**Рекомендации для администратора.** Опиши, что перед началом работы рекомендуется:
- проверить подключение к базе данных и доступность веб-сервера;
- убедиться в корректности настроек текущей недели, дня и времени в schedule_settings;
- проверить полноту данных в schedule_all для всех учебных групп;
- перед началом семестра актуализировать расписание для первой и второй недели;
- контролировать учётные записи пользователей и их статусы;
- выполнять резервное копирование базы данных перед массовым обновлением расписания.

**Рекомендации для преподавателя.** Укажи:
- регулярно проверять своё расписание в режиме просмотра по преподавателю;
- использовать переключение дня и недели для планирования нагрузки;
- обращаться к администратору при обнаружении ошибок в данных о занятиях;
- использовать функцию «Поделиться расписанием» для информирования студентов.

**Рекомендации для студента.** Опиши:
- выбирать свою учебную группу при первом посещении системы;
- регулярно просматривать расписание с учётом чередования недель;
- обращать внимание на блок текущей и следующей пары;
- добавлять группу в избранное для быстрого доступа;
- использовать надёжный пароль и защиту учётных данных при авторизации;
- действия при обнаружении несоответствий (обращение к администратору или преподавателю).

**Рекомендации для технического специалиста.** Укажи:
- регулярное резервное копирование базы данных с таблицами schedule_all и schedule_settings;
- контроль журналов ошибок и состояния серверных сервисов Apache и MySQL;
- проверку восстановления данных из резервной копии;
- контроль производительности при росте числа групп и занятий;
- мониторинг работы программного интерфейса и webhook мессенджера;
- своевременное обновление времени в schedule_settings при смене учебного периода.

**Таблица рекомендаций.** В тексте добавь ссылку:
**«Рекомендации по использованию информационной системы электронного расписания приведены в таблице 12».**

Таблицу 12 в текст полностью не вставляй. Считай, что она подготовлена отдельным `xlsx` файлом.

**Итог раздела.** Подведи вывод, что соблюдение рекомендаций повышает устойчивость эксплуатации, снижает число ошибок при актуализации расписания и повышает качество работы информационной системы.

### Общие требования

- Пиши академическим стилем, сплошными абзацами, по `rules.md`.
- Нумерацию источников и ссылки в квадратных скобках оформляй в том порядке, который задает пользователь.
